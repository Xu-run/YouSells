# YouSells P0 开发任务分配与进度管理方案

基于 `1 名队长统筹基础设施与验收 + 2 名后端并行开发 + 2 名前端并行开发` 的协作模式  
日期：`2026 年 5 月 18 日`

---

## 一、P0 产品定位与开发总览

### 1.1 P0 目标

`YouSells` 的 `P0` 不是做完整平台，而是先把内部业务中台最核心的一条主流程打通：

`登录 -> 首页看板 -> 客户总表 -> 客户详情 -> 跟进记录 -> 公共安排 -> 日报/周报 -> 话术库`

P0 目标是：

- 把项目基础设施搭起来
- 把权限和数据口径先统一
- 把核心业务模块全部跑通
- 把自动化测试流程强制带起来
- 为后续 `P1 / P2` 扩展留稳定基础

### 1.2 P0 范围内功能

模块 | 功能 | 独立性
--- | --- | ---
模块 A | 登录与权限基础 | 共享基础
模块 B | 首页看板 | 依赖共享接口
模块 C | 客户总表 + 客户详情 | 核心业务
模块 D | 跟进记录 | 依赖客户模块
模块 E | 公共安排区 | 相对独立
模块 F | 日报 / 周报 | 相对独立
模块 G | 话术库 | 相对独立
公共模块 | 工程基线、数据库设计、接口规范、测试规范、GitHub 协作、部署基线 | 共享

### 1.3 P0 不做功能

以下内容不属于 `P0` 范围：

- 月报
- 复盘总结区
- 匿名意见箱
- 工作聊天室
- 知识分享区
- 博客 / 日常分享区
- 微服务拆分
- JWT 正式切换
- 对外开放 API

### 1.5 当前协作原则

- `秦梓源` 负责每个阶段的基础设施准备和最终测试验收
- 模块开发者完成代码后，必须立刻写模块自动化测试
- 所有页面相关改动，必须明确提醒开发者做前后端联调
- 开发前先定需求、开发规划、开发日志，再写业务代码
- 每个模块都必须有负责人、依赖项、验收标准、状态管理

## 九、当前阶段结论

当前 `YouSells` 的 `P0` 开发，不是简单按页面拆活，而是按 `基础设施 -> 权限与数据库 -> API 基线 -> 前后端并行 -> 联调 -> 测试验收` 这条完整链路推进。

当前马上该推进的事只有 4 件：

1. 定数据库表设计
2. 定接口规范
3. 初始化前后端工程
4. 正式进入 `P0` 各模块并行开发

当前这 4 件中，前 3 件已经完成，下一步应正式进入：

1. 志明负责 `BE-002 ~ BE-004`
2. 哲涛负责 `BE-005 ~ BE-007`
3. 嘉诚负责 `FE-002 ~ FE-004`
