# YouSells 本地启动与验收指南

日期：`2026-05-19`

## 1. 本地依赖

成员本机需要：

- `Docker Desktop`
- `Docker Compose`
- `JDK 21`
- `Maven`
- `Node.js`
- `npm`

先检查：

```powershell
docker --version
docker compose version
java -version
mvn -version
node -v
npm -v
```

`java -version` 和 `mvn -version` 都必须指向 Java 21。

如果 `java -version` 是 21，但 `mvn -version` 显示 Java 11，先切换：

```powershell
$env:JAVA_HOME="D:\develop\jdk21"
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn -version
```

## 2. 基础服务

项目根目录执行：

```powershell
docker compose up -d
```

查看状态：

```powershell
docker compose ps
```

默认端口：

- `MySQL: 13306`
- `Redis: 16379`
- `Backend: 8080`
- `Frontend: 5173`

MySQL 默认连接：

- Host: `localhost`
- Port: `13306`
- Database: `yousells`
- Username: `root`
- Password: `root`

Redis 默认连接：

- Host: `localhost`
- Port: `16379`

## 3. 数据库初始化

MySQL 首次初始化时会执行：

- `yousells-server/src/main/resources/db/01_schema_p0.sql`
- `yousells-server/src/main/resources/db/02_seed_p0.sql`

校验：

```powershell
docker exec -it yousells-mysql mysql -uroot -proot -e "SHOW DATABASES;"
docker exec -it yousells-mysql mysql -uroot -proot -D yousells -e "SHOW TABLES;"
docker exec -it yousells-redis redis-cli ping
```

Redis 返回 `PONG`，MySQL 能看到 `yousells` 和表，即基础服务正常。

## 4. 启动后端

进入后端目录：

```powershell
cd yousells-server
mvn spring-boot:run
```

健康检查：

```powershell
curl http://localhost:8080/actuator/health
```

## 5. 启动前端

进入前端目录：

```powershell
cd yousells-web
cmd /c npm install
cmd /c npm run dev
```

浏览器打开：

```text
http://localhost:5173/login
```

演示账号：

- `admin / admin123`
- `member / member123`

## 6. 验收命令

后端测试：

```powershell
cd yousells-server
mvn test
```

前端构建：

```powershell
cd yousells-web
cmd /c npm run build
```

页面验收：

- 登录成功后进入 `/dashboard`
- 刷新后仍保持登录态
- 控制台无项目自身 error
- 如果看到 `aegisInject`、`hybridaction`、`daxuesoutijiang.com/api/log`，优先判断为浏览器插件/外部注入脚本，不作为项目错误处理

## 7. 常见问题

端口被占用：

- 检查 `13306 / 16379 / 8080 / 5173`
- 必须改端口时，同时改 `docker-compose.yml`、环境变量和文档

SQL 没自动执行：

- 可能是 MySQL 数据卷已存在
- 确认要重置时执行：

```powershell
docker compose down -v
docker compose up -d
```

登录后仍 401：

- 确认前端 `.env.development` 是 `VITE_API_BASE_URL=/api`
- 确认请求头有 `Authorization: Bearer <token>`
- 确认后端使用最新 JWT 代码
- 不要回退到 Session/Cookie 方案

Maven 用错 JDK：

- `mvn -version` 如果显示 Java 11，先切换 `JAVA_HOME`
- 后端项目要求 JDK 21
