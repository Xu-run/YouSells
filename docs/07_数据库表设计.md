# 数据库表设计

## 1. 设计目标

`YouSells` 的 `P0` 数据库设计目标不是追求复杂，而是先满足下面 4 件事：

1. 支撑 `登录 -> 客户管理 -> 跟进记录 -> 公共安排 -> 日报周报 -> 话术库` 主流程
2. 保证字段口径统一，避免后面每个人写出不同版本的数据结构
3. 方便后端模块拆分开发
4. 为后续 `P1 / P2` 扩展预留空间，但不提前做重

当前数据库默认使用：

- `MySQL 8`
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_0900_ai_ci`

## 2. 命名规范

### 表命名

- 全部使用 `snake_case`
- 表名用复数语义或稳定业务名
- 不使用拼音缩写

### 字段命名

- 全部使用 `snake_case`
- 主键统一为 `id`
- 创建时间统一为 `created_at`
- 更新时间统一为 `updated_at`
- 创建人统一为 `created_by`
- 更新人统一为 `updated_by`
- 逻辑删除字段统一为 `is_deleted`

### 主键建议

- `P0` 阶段统一使用 `bigint`
- 主键生成可以用：
  - MySQL 自增
  - 或雪花 ID

如果你们想减少复杂度，`P0` 建议直接用自增 `bigint`。
