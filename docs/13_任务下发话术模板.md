# YouSells 任务下发话术模板

日期：`2026 年 5 月 19 日`

这份文档是给你直接发给队员用的。  
分为：

- 群发总说明
- 私发给每个人的单独任务话术

你可以直接复制发，也可以按你自己的语气微调。

---

## 1. 群发总说明

大家这两天正式开始推进 `YouSells` 的 `P0` 开发。

这一轮不是随便写页面和接口，而是按正式项目流程推进。我们现在基础设施、数据库基线、API 基线、任务分工和架构边界都已经定下来了，后面大家按分工各自推进，不要跨模块乱改。

这次统一要求先看 3 份文档，再开工：

- `docs/11_后端架构收口与协作边界规范.md`
- `docs/06_P0开发任务分配方案.md`
- `docs/05_开发流程与日志规范.md`

前端同学再额外看：

- `docs/08_API接口基线.md`

统一执行规则：

- 每个人只改自己独占目录
- 开发前先明确任务范围，不要上来就开写
- 写完模块代码立刻补自动化测试
- 如果接口字段有变化，要先同步文档，再做联调
- 不允许顺手改别人模块，不允许一个 PR 混很多无关改动

后面如果发现必须改到别人的文件，不要自己直接动，先来找我，我来判断是补共享基线还是拆新文件。

这一轮大家的目标不是“先写出来”，而是“按边界把自己的模块稳稳做完”。

## 2. 发给志明的话术

志明，你这轮主要负责后端客户域和跟进域。

你只负责这些范围：

- `modules/customer/**`
- `modules/followup/**`
- 对应测试目录
- 对应 `mapper` 目录

默认不要改：

- `common/**`
- `dashboard/**`
- `task / report / script` 相关目录

你这边任务顺序按这个来：

1. 客户域持久化底座
2. 客户总表接口真实化
3. 客户详情接口真实化
4. 跟进模块真实化与客户联动

你这一轮的核心要求：

- 先把 `entity / mapper / convert` 补起来
- 客户域和跟进域尽量在你自己目录内闭环
- 跟进新增后，要把客户的 `next_follow_at` 和最新反馈联动更新好
- 写完每一步都要立刻补测试

你开工前先看这几份：

- `docs/11_后端架构收口与协作边界规范.md`
- `docs/06_P0开发任务分配方案.md`
- `docs/05_开发流程与日志规范.md`

如果中间发现需要动共享文件或者别人目录，先别自己改，直接来找我。

## 4. 发给嘉诚的话术

嘉诚，你这轮主要负责前端登录页、首页看板、客户总表这三块。

你只负责这些范围：

- 登录页
- 首页看板
- 客户总表页
- 对应组件目录
- 对应 API 文件
- 对应类型文件

默认不要改：

- `router/index.ts`
- `layouts/AppLayout.vue`
- `components/app/**`
- 许润负责的详情、跟进、任务、日报、话术相关页面

你这边任务顺序按这个来：

1. 登录页联调收口
2. 首页看板组件化
3. 客户总表组件化

你这一轮的核心要求：

- 页面尽量拆成清晰组件，不要把逻辑都堆在 view 里
- 只在你自己的页面、组件、API、类型文件范围内改
- 接口一旦发现字段不够，不要自己去改后端共享文件，先反馈给我
- 每个页面做完都要自己联调一遍

你开工前先看这几份：

- `docs/06_P0开发任务分配方案.md`
- `docs/05_开发流程与日志规范.md`
- `docs/08_API接口基线.md`

路由和共享壳子我来统一收，不需要你在 PR 里顺手改这些入口文件。

## 6. 你自己发任务时的建议

你实际发的时候建议顺序是：

1. 先群发总说明
2. 再分别私发个人任务
3. 最后让每个人回一句“已看完文档 + 明确自己目录范围”

你这样做的好处是：

- 先统一规则
- 再分个人任务
- 最后确认大家真的知道自己能改什么、不能改什么

这样后面会少很多低级冲突。
