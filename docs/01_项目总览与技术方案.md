# YouSells 项目总览与技术方案

日期：`2026-05-19`

## 1. 项目定位

`YouSells` 是给团队内部使用的客户管理与协作平台，第一阶段不是做大而全的团队社区，而是先把每天会真实使用的业务主流程跑通。

P0 主流程：

`登录 -> 首页看板 -> 客户总表 -> 客户详情 -> 跟进记录 -> 公共安排 -> 日报/周报 -> 话术库`

## 2. 当前使用对象

- 团队内部成员
- 团队负责人 / 管理员
- 目标使用人数：`50-100`
- 使用场景：客户跟进、公共安排、日报周报、话术沉淀、团队协作

## 3. P0 范围

P0 必做：

- JWT 登录与当前用户识别
- 首页看板聚合
- 客户总表
- 客户详情
- 跟进记录
- 公共安排
- 日报 / 周报
- 话术库
- 本地 MySQL / Redis 开发环境
- 后端自动化测试基线
- 前后端联调基线

P0 不做：

- 月报
- 复盘总结区
- 匿名意见箱
- 工作聊天室
- 知识分享区
- 博客 / 日常分享区
- 微服务拆分
- refresh token / Redis 黑名单 / 多端登录踢出
- 对外开放 API

## 4. 技术栈

后端：

- `JDK 21`
- `Spring Boot 3.5`
- `Spring Security`
- `JWT Bearer Token`
- `MyBatis-Plus`
- `MySQL 8`
- `Redis`

前端：

- `Vue 3`
- `TypeScript`
- `Vite`
- `Pinia`
- `Vue Router`
- `Element Plus`
- `Axios`

部署方向：

- P0 本地：`Docker Compose` 启动 MySQL + Redis，前后端本地运行
- 第一版上线：`Nginx + Spring Boot 单体 + MySQL + Redis`
- 后续增强：`refresh token + Redis 黑名单`
- 再后续：`网关 + 认证中心 + 业务服务拆分`

## 6. 鉴权方案

当前采用轻量 JWT：

- 登录接口返回 `accessToken`
- 前端存入 `localStorage`
- 请求头统一带 `Authorization: Bearer <token>`
- 后端 `JwtAuthenticationFilter` 解析 token 并写入 `SecurityContext`
- `/api/auth/me` 用于刷新页面后的登录态恢复

当前不做：

- refresh token
- Redis token 黑名单
- token 主动失效管理
- 多端登录踢出

这些是后续正式上线前的增强项，不影响 P0 本地联调。

## 7. 关键原则

- 先做 P0
- 先打通主流程
- 先保证每天真能用
- 先让模块边界清楚
- 不一开始做太重
- 为后续扩展留空间，但不提前透支复杂度
