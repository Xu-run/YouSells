## 1. 需求摘要

将 `ScriptLibraryServiceImpl` 从硬编码样例数据切换为真实 MySQL 持久化，涉及两张表：`script_categories` 和 `scripts`。

## 2. 涉及数据库表

### 2.1 `script_categories`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK AUTO | 主键 |
| category_code | VARCHAR(64) | NOT NULL, UNIQUE | 分类代码 |
| category_name | VARCHAR(64) | NOT NULL | 分类名称 |
| sort_order | INT | NOT NULL DEFAULT 0 | 排序 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |
| is_deleted | TINYINT(1) | NOT NULL DEFAULT 0 | 逻辑删除 |

> ⚠️ 此表**没有** `created_by` / `updated_by` 字段，Entity 不能继承 `BaseEntity`。

### 2.2 `scripts`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK AUTO | 主键 |
| category_id | BIGINT | NOT NULL | 分类ID，外键到 script_categories |
| title | VARCHAR(128) | NOT NULL | 标题 |
| content | TEXT | NOT NULL | 话术内容 |
| applicable_scene | VARCHAR(255) | NULL | 适用场景 |
| status | VARCHAR(32) | NOT NULL | 状态 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |
| created_by | BIGINT | NULL | 创建人 |
| updated_by | BIGINT | NULL | 更新人 |
| is_deleted | TINYINT(1) | NOT NULL DEFAULT 0 | 逻辑删除 |

> ✅ 此表**有** `created_by` / `updated_by` 字段，Entity **可继承** `BaseEntity`。

- 无 entity / mapper / convert / XML

## 6. 详细设计

### 6.1 Entity 设计

#### ScriptCategoryEntity
- 不继承 BaseEntity（表无 created_by/updated_by）
- @TableName("script_categories")
- 字段：id(@TableId AUTO), categoryCode, categoryName, sortOrder
- 审计：createdAt(fill=INSERT), updatedAt(fill=INSERT_UPDATE), isDeleted(@TableLogic)
- 用 @TableField(exist = false) 声明 createdBy/updatedBy 以兼容全局 MetaObjectHandler

#### ScriptEntity
- 继承 BaseEntity（表有 created_by/updated_by）
- @TableName("scripts")
- 新增字段：categoryId, title, content, applicableScene, status
- 审计字段由 BaseEntity 继承

### 6.2 Mapper 设计

#### ScriptCategoryMapper
- extends BaseMapper<ScriptCategoryEntity>
- 无自定义方法，Service 层通过 LambdaQueryWrapper 查询

#### ScriptMapper
- extends BaseMapper<ScriptEntity>
- `IPage<ScriptVo> selectPageWithCategory(Page, categoryId, keyword)`
  - LEFT JOIN script_categories ON category_id
  - 动态 SQL：`<if test="categoryId != null">` 分类筛选
  - 动态 SQL：`<if test="keyword != null and keyword != ''">` 标题/内容模糊搜索
  - 按 updated_at DESC 排序
  - Constructor 映射 ScriptVo 记录

### 6.3 Convert 设计

```
ScriptConvert（工具类，不可实例化）
  toEntity(ScriptCreateRequest) → ScriptEntity（默认 status = "ENABLED"）
  applyUpdate(ScriptEntity, ScriptUpdateRequest) → void（默认 status = "ENABLED"）
```

### 6.4 ServiceImpl 改造

| 方法 | 改造逻辑 |
|------|----------|
| `listCategories` | `scriptCategoryMapper.selectList(QueryWrapper: is_deleted=0, order by sort_order)` → map entity → ScriptCategoryVo |
| `pageScripts` | `scriptMapper.selectPageWithCategory(Page.of(page, pageSize), categoryId, keyword)` → PageResponse |
| `getScript` | `scriptMapper.selectById(id)` → 不存在抛 NOT_FOUND → `scriptCategoryMapper.selectById(categoryId)` 取分类名 → 组装 ScriptVo |
| `createScript` | `ScriptConvert.toEntity(request)` → `scriptMapper.insert(entity)` → 返回 `entity.getId()` |
| `updateScript` | `scriptMapper.selectById(id)` → 不存在抛 NOT_FOUND → `ScriptConvert.applyUpdate(entity, request)` → `scriptMapper.updateById(entity)` |
