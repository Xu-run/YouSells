## 3. 字段一致性检查

### 3.1 命名规范

全部遵循 Java camelCase，序列化后前端解析一致：

- `taskTitle`, `ownerDisplayName`, `assistantDisplayName`, `dueAt`, `nextAction`
- `reportDate`, `userDisplayName`, `todayWork`, `tomorrowPlan`
- `weekKey`, `weeklySummary`, `nextWeekPlan`
- `categoryId`, `categoryName`, `applicableScene`, `updatedAt`

无蛇形命名混入。✅

### 3.2 日期/时间类型

| VO | 字段 | Java 类型 | JSON 格式 |
|----|------|-----------|-----------|
| TaskBoardItemVo | dueAt | LocalDateTime | ISO `"2026-05-20T12:00:00"` |
| DailyReportVo | reportDate | LocalDate | ISO `"2026-05-20"` |
| ScriptVo | updatedAt | LocalDateTime | ISO `"2026-05-20T12:00:00"` |

全部 ISO 标准格式，前端可直接解析。✅

### 3.3 分页结构统一

  "message": "success",
  "data": {
    "list": [],
    "page": 1,

三个模块完全一致。✅

### 3.4 错误响应统一

| 场景 | 错误码 | 异常类 |
|------|--------|--------|
| 资源不存在 | NOT_FOUND (4004) | BusinessException |
