## 问题清单

### #1 DashboardControllerIntegrationTest 全量测试失败

- **发现日期**：`2026-05-20`
- **所属模块**：`dashboard`（负责人：秦梓源）
- **严重程度**：中（不影响 P0 业务模块测试，但阻塞全量 `mvn test`）
- **状态**：🔴 待修复

**现象**：

```
DashboardControllerIntegrationTest.shouldReturnDashboardOverviewAfterLogin:59
Status expected:<200> but was:<500>
```

**原因分析**：

`DashboardControllerIntegrationTest` 使用 `@SpringBootTest` + `@ActiveProfiles("test")` 启动完整 Spring 上下文。`application-test.yml` 中配置了 Redis 连接：

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
```

但本地环境中 Redis 服务未启动（Docker 未安装），导致 Redis 自动配置尝试连接失败，login 请求返回 500。

**影响范围**：
- 全量 `mvn test` 会失败（15 个测试中 1 个失败）
- 不影响各模块独立单元测试（`TaskBoardServiceTest` 等不依赖 Spring 上下文的测试正常通过）
- 不影响 `mvn spring-boot:run` 启动（使用 dev profile，可正常连接 Docker MySQL/Redis）

**建议修复方向**：
- 方案 A：测试 profile 中排除 Redis 自动配置，或使用 Testcontainers
- 方案 B：在 CI 环境中确保 Redis 可用
- 方案 C：`DashboardControllerIntegrationTest` 中 mock 掉 Redis 相关 bean
