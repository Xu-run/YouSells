#### 组件 1: `CustomerFilterBar.vue`

**职责**： 筛选条件输入 + 搜索/重置按钮

| Props | 类型 | 说明 |
|---|---|---|
| `modelValue` | `CustomerQuery` | 当前查询参数 |
| `loading` | `boolean` | 是否加载中 |

| Emits | 参数 | 说明 |
|---|---|---|
| `update:modelValue` | `CustomerQuery` | 查询参数变更 |
| `search` | — | 点击搜索 |
| `reset` | — | 点击重置 |

**筛选项：**

| 字段 | Element Plus 组件 | 选项/说明 |
|---|---|---|
| `keyword` | `el-input` | placeholder="搜索姓名/专业"，支持回车触发搜索 |
| `grade` | `el-select` | 全部 / 大一 / 大二 / 大三 / 大四 |
| `progress` | `el-select` | 全部 / 职规 / 技术栈 / 课程 |
| `intent` | `el-select` | 全部 / 很稳 / 可跟 / 观望 / 冷淡 |

**交互逻辑：**
- 任意筛选条件变化时 `emit("update:modelValue", newQuery)`
- 点击搜索时自动将 `page` 重置为 1，然后 `emit("search")`
- 点击重置时清空所有条件，`page` 回到 1，`emit("update:modelValue")` + `emit("reset")`
- loading 期间搜索按钮 disabled

**模板结构：**

    el-select  (grade)
    el-select  (progress)

#### 页面: `CustomerListView.vue`

- `customers`: `CustomerListItem[]`

- `query`: 筛选条件（`CustomerQuery`）
- `createDialogVisible`: 新建弹窗开关

#### 页面 1: `DailyReportView.vue`

**数据流：**
1. `onMounted` → 默认选中今天 → `loadReport(today)` → `GET /api/reports/daily?date=today`
2. 返回 `null`: 今天还没写 → 显示自动聚合数据 + 空表单
3. 返回有数据: 今天已写 → 展示完整日报（自动数据 + 人手写内容），今天内可修改
4. 切换日期 → `loadReport(selectedDate)` → 展示历史日报（只读）
5. 提交 → `POST /api/reports/daily` → 成功后切换到已提交状态

**状态：**

| 状态 | 条件 | 表现 |
|---|---|---|
| loading | 请求中 | 自动数据区占位 `—`，表单 disabled |
| empty（今天还没写） | 返回 null && 选的是今天 | 自动数据加载，表单可编辑，显示提交按钮 |
| submitted（今天已提交） | 返回有数据 && 选的是今天 | 自动数据 + 人手写内容，当天内可修改 |
| readonly（历史） | 选的是过去日期 | 全部只读，不显示修改/提交按钮 |
| error | 接口报错 | 显示失败提示 |

**模板结构：**
```
PageSection(title="日报" description="系统自动汇总今日数据，只需填写小结/问题/计划")
  .report-layout
    ReportHistory(v-model:current-date="selectedDate" type="daily" @change)

    div(v-if="loading") 加载中...
    div(v-else-if="error") 日报加载失败

    div(v-else)
      // 自动数据区（今天还没写时也需要展示自动数据）
      ReportAutoData(:data="autoData" type="daily" :loading)

      // 已提交：展示已填内容
      div(v-if="submitted && !editable")
        el-alert(type="info" :closable="false") 此日报已于 {{ formatDate(report.createdAt) }} 提交

      // 表单区（可编辑状态下显示）
      ReportForm(v-if="!submitted || editable"
        v-model="form" :loading="submitLoading"
        :submitted :editable
        @submit)

      // 已提交且不可改：只读展示
      .readonly-view(v-else)
        el-descriptions(:column="1")
          el-descriptions-item(label="今日小结") {{ report.summary }}
          el-descriptions-item(label="遇到问题") {{ report.issues || '—' }}
          el-descriptions-item(label="明天计划") {{ report.tomorrowPlan }}
```

## 16. 进度管理

### 状态词汇

- 🔴 未开始
- 🔵 进行中
- ⛔ 阻塞中
- ✅ 已完成

### ⛔ 阻塞定义

任务在以下情况下标记为⛔ 阻塞中：
- 前置任务未完成
- 外部依赖（API、人员、决策）超过 24 小时无法推进

阻塞时必须立即通知队长，说明：阻塞原因 + 需要什么解除 + 可以先做什么。

### 每日更新格式

```
昨日完成：
今日计划：
阻塞（如有）：
PR / 分支链接：
测试结果：
```

### 完成定义

- 代码或产出完成
- 测试通过
- 文档或交接说明更新
- 审查者无需猜测即可验证

---

## 17. PR 规则

- 分支名含任务 ID（如 `feature/BE-102-customer-rebuild`）
- PR 标题含任务 ID + 模块
- PR 描述列出变更范围、测试运行结果、UI 变更截图
- 不得编辑他人独占目录
- 共享文件需队长审查
- 合并顺序遵循依赖图
