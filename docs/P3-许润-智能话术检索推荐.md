## 一、模块目标

在写跟进时，AI 根据当前客户上下文自动推荐最匹配的话术。同时增强话术库搜索能力——输入自然语言问题（如"怎么跟冷淡的客户破冰"），AI 语义匹配相关话术。

1. **跟进时推荐**：在 FollowUpCreateForm 旁新增"AI 话术推荐"侧栏
2. **语义搜索**：话术库搜索框支持自然语言查询
3. **一键引用**：推荐的话术可一键填入跟进内容

## 二、前后端接口契约

### 2.1 POST `/api/scripts/recommend`

**请求体：**
```json
{
  "customerId": 1,
  "context": {
    "progress": "技术栈",
    "intent": "可跟",
    "latestFollowUpContent": "客户表示对Python后端开发感兴趣，但担心学习难度"
  }
}
```

**TypeScript 类型：**
```ts
// src/types/script.ts 中扩展
export interface ScriptRecommendRequest {
  customerId: number;
  context: {
    progress: string;
    intent: string;
    latestFollowUpContent?: string;
  };
}

export interface ScriptRecommendItem {
  script: ScriptItem;
  relevanceScore: number;      // 0-100
  relevanceReason: string;     // "该话术适用于技术栈阶段的客户"
}

// src/api/script.ts 中新增
export async function recommendScripts(data: ScriptRecommendRequest): Promise<ScriptRecommendItem[]> {
  const response = await request.post<ApiResponse<ScriptRecommendItem[]>>("/scripts/recommend", data);
  return response.data.data;
}
```

**响应示例：**
```json
{
  "code": 200,
  "data": [
    {
      "script": {
        "id": 12,
        "title": "技术栈犹豫型客户破冰话术",
        "content": "同学你好，其实很多同学在入门阶段都会有类似的顾虑...",
        "category": "破冰"
      },
      "relevanceScore": 92,
      "relevanceReason": "该客户处于技术栈阶段且表现出犹豫，此话术针对此场景设计"
    },
    {
      "script": {
        "id": 8,
        "title": "Python课程价值陈述",
        "content": "Python是当下最通用的编程语言...",
        "category": "课程介绍"
      },
      "relevanceScore": 78,
      "relevanceReason": "客户明确提及Python，话术提供了对应的价值陈述"
    }
  ]
}
```

### 2.2 GET `/api/scripts/search`

**查询参数：** `keyword`（自然语言查询字符串）、`page`、`pageSize`

**现有行为：** 基于 SQL LIKE 进行关键词匹配
**P3 增强：** 支持自然语言查询时调 AI 进行语义理解和匹配排序

当 `keyword` 包含自然语言特征（问句、超过一定长度）时，后端将 keyword 和话术库交给 LLM，让 LLM 选出最相关的 N 条。

**响应格式不变**——兼容现有前端分页列表。

### 3.2 话术推荐核心逻辑

```
1. 收到请求 { customerId, context }
2. 查询话术库全部话术（话术库数据量不大，预计 50-200 条）
3. 组装 Prompt：
   - 客户上下文（进度 + 意向 + 最近跟进内容）
   - 话术库列表（id + title + category + content 摘要）
   - 要求 LLM 选出最匹配的 3-5 条，给出匹配理由和评分
4. 调 AiService.generate(prompt, List<ScriptRecommendItem>.class)
5. 返回推荐结果
```

### 3.3 语义搜索核心逻辑

```
1. 判断 keyword 是否为自然语言查询（长度 > 10 或包含问号/"怎么"/"如何"等）
2. 如果是自然语言查询 →
   a. 获取全部话术列表
   b. Prompt = "用户问题：" + keyword + "话术列表：" + scripts
   c. 让 LLM 选出最相关的 N 条，返回 id 列表
   d. 用 id 列表去数据库查完整话术 → 返回分页结果
3. 如果是关键词查询 → 走原有 SQL LIKE 逻辑
```

### 3.4 Prompt 设计提示

System prompt：
> "你是话术库检索专家。根据当前客户的情况和沟通上下文，从话术库中挑选最适合这个场景的话术。评估标准：
> 1. 话术内容与客户当前进度的匹配度
> 2. 话术风格与客户沟通场景的适配性
> 3. 话术是否解决了当前沟通中的具体障碍
> 返回 JSON 数组，每项包含 scriptId、relevanceScore(0-100)、relevanceReason。"

### 3.5 测试场景

| 场景 | 输入 | 期望输出 |
|---|---|---|
| 正常推荐 | customerId=1, 有完整上下文 | 返回 3-5 条推荐，每项有评分和理由 |
| 话术库为空 | 无话术 | 返回空数组 |
| 客户无跟进记录 | customerId=3, 0条跟进 | 基于进度和意向推荐，latestFollowUpContent 为空 |
| 语义搜索：自然语言 | keyword="怎么跟冷淡的客户破冰" | LLM 语义匹配，返回相关话术 |
| 语义搜索：关键词 | keyword="Python" | 走原 SQL LIKE 逻辑 |

### 4.2 AiScriptPanel 组件规格

**Props：**
| 属性 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `customerId` | `number` | 是 | 客户 ID |
| `customerContext` | `{ progress: string; intent: string; latestContent?: string }` | 是 | 客户上下文 |
| `visible` | `boolean` | 是 | 控制显示/隐藏 |

**Emits：**
| 事件 | 参数 | 说明 |
|---|---|---|
| `close` | — | 关闭面板 |
| `insert` | `content: string` | 用户选择话术后，将内容插入跟进表单 |

**Template 结构（伪代码）：**
```html
<el-drawer v-model="visible" title="AI 话术推荐" direction="rtl" size="400px">
  <!-- 加载中 -->
  <div v-if="loading" class="loading-state">
    <el-icon class="is-loading"><Loading /></el-icon>
    <p>AI 正在分析客户情况，为你推荐最合适的话术...</p>
  </div>

  <!-- 推荐列表 -->
  <div v-else-if="recommendations.length > 0" class="recommend-list">
    <div v-for="item in recommendations" :key="item.script.id" class="recommend-item">
      <div class="item-header">
        <el-tag size="small">{{ item.script.category }}</el-tag>
        <span class="score">匹配度 {{ item.relevanceScore }}%</span>
      </div>
      <h4>{{ item.script.title }}</h4>
      <p class="item-reason">{{ item.relevanceReason }}</p>
      <div class="item-content">{{ item.script.content }}</div>
      <el-button type="primary" size="small" @click="$emit('insert', item.script.content)">
        引用此话术
      </el-button>
    </div>
  </div>

  <!-- 无推荐 -->
  <EmptyState v-else title="暂无推荐" description="话术库中暂未找到匹配当前场景的话术" />

  <!-- 刷新 -->
  <template #footer>
    <el-button :loading="loading" @click="loadRecommendations">换一批</el-button>
  </template>
</el-drawer>
```

**状态覆盖：**

| 状态 | 展示 |
|---|---|
| 加载中 | 骨架屏 + "AI 分析中..." |
| 有推荐 | 推荐列表，每项有评分/理由/内容/引用按钮 |
| 无推荐 | EmptyState |
| 加载失败 | Alert + 重试 |

## 六、遇到问题怎么办

1. **话术库数据量太小（<20条）**：推荐效果可能弱，可以先用种子数据丰富话术库
2. **推荐不够精准**：检查 Prompt 中是否提供了足够的客户上下文，试着手动加几条跟进内容看效果
3. **语义搜索和关键词搜索判断不准**：用简单的规则判断自然语言（查问号/疑问词），后续可优化
4. **前端侧栏太窄显示不完整**：用 el-drawer 可调整宽度，建议 400-480px
