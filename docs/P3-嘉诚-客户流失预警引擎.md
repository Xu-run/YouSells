## 一、模块目标

构建一个自动化的流失预警系统，AI 自动识别有流失风险的客户，分级预警并提供干预建议。

1. **风险评分引擎**：多维度分析（沉默天数、意向变化、跟进频率、最后跟进情感）→ 输出风险等级
2. **Dashboard 风险卡片**：首页新增"流失风险"面板，展示高风险客户
3. **主动通知**：客户进入高风险自动通知负责人
4. **干预建议**：每个风险客户附带 AI 生成的"建议措施"

### 2.1 GET `/api/dashboard/churn-risks`

**说明：** 获取流失风险客户列表。Dashboard 页面调用。

**TypeScript 类型：**
```ts
// src/types/churn.ts（新建）
export interface ChurnRiskItem {
  customerId: number;
  customerName: string;
  riskLevel: "high" | "medium" | "low";
  riskScore: number;                // 0-100，越高越危险
  riskFactors: string[];            // ["沉默超过7天", "意向下降", "未跟进"]
  suggestion: string;               // AI 生成的干预建议
  silentDays: number;
  lastFollowUpAt: string | null;
  evaluatedAt: string;
}

export interface ChurnRiskResponse {
  highRisk: ChurnRiskItem[];
  mediumRisk: ChurnRiskItem[];
  total: number;
  evaluatedAt: string;
}

// src/api/dashboard.ts 中新增
export async function fetchChurnRisks(): Promise<ChurnRiskResponse> {
  const response = await request.get<ApiResponse<ChurnRiskResponse>>("/dashboard/churn-risks");
  return response.data.data;
}
```

### 3.2 核心评分逻辑

```
评估流程（可被定时任务或手动触发）:
1. 查询所有活跃客户（status = ACTIVE）
2. 对每个客户计算流失风险：
   a. 规则引擎初筛：
      - 沉默天数 > 14  → 基础分 +40
      - 沉默天数 > 7   → 基础分 +20
      - 意向从"很稳"→"可跟/观望"  → +15
      - 意向从任何 →"冷淡"         → +25
      - 最后跟进内容包含"联系不上/无人接/不回复" → +20
      - 30天内跟进次数为 0 → +20
      - 有下一步计划但7天内未执行 → +10
   b. 总分 >= 60 → 高风险
      总分 >= 30 → 中风险
      总分 < 30 → 低风险
   c. 中/高风险客户 → 调 LLM 做深度分析：
      - 输入：客户基本信息 + 跟进历史 + 规则评分结果
      - 输出：风险因素列表 + 干预建议文案
3. 结果写入 ai_churn_risk 表
4. 高风险客户触发通知
```

### 3.4 定时任务

```java
@Component
public class ChurnEvaluationJob {
    // 每天上午 8:00 执行一次
    @Scheduled(cron = "0 0 8 * * ?")
    public void evaluate() {
        customerService.evaluateChurnRisks();
    }
}
```

### 3.5 测试场景

| 场景 | 输入 | 期望输出 |
|---|---|---|
| 高风险客户 | 沉默15天+意向下降+联系不上 | riskLevel=high, riskScore >= 60 |
| 正常客户 | 沉默1天+意向稳定+有跟进 | 低风险，不调 LLM |
| 无跟进记录的客户 | 注册7天无跟进 | riskLevel=medium |
| 规则边缘 | 沉默6天，总分29 | riskLevel=low |
| LLM 分析失败 | 得分>=30但LLM超时 | 使用规则结果，suggestion 用默认文案 |

## 六、遇到问题怎么办

1. **规则评分不合理**（太多/太少高风险）：调整阈值——改规则引擎的分数门槛，不是改 LLM
2. **LLM 分析太慢**（评估很多客户时）：只对中高风险客户调 LLM，低风险只做规则判断
3. **通知太频繁**：同一客户一天内不重复通知，加去重逻辑
4. **风险评估与沉默客户面板重叠**：直接替换——流失预警是沉默客户的升级版
5. **卡住超过半天**：群里说明，@梓源
